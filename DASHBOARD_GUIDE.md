# Dashboard & Extension Usage Guide

이 문서는 ReactPerf 대시보드와 크롬 확장 프로그램을 사용하여 데이터를 수집하고 분석하는 방법을 설명합니다.

## 1. 서비스 구성

- **대시보드 앱 (Next.js)**: 수집된 성능 데이터를 저장하고 시각화하는 중앙 제어 센터입니다.
- **크롬 확장 프로그램 (Chrome Extension)**: 웹 브라우저에서 React 앱의 렌더링 지표와 API 요청 데이터를 캡처하여 대시보드로 전송합니다.

## 2. 사용 단계

### 1단계: 크롬 확장 프로그램 설치

1.  크롬 브라우저에서 `chrome://extensions/` 주소로 이동합니다.
2.  우측 상단의 **'개발자 모드'**를 활성화합니다.
3.  **'압축해제된 확장 프로그램을 로드합니다'** 버튼을 클릭합니다.
4.  현재 프로젝트 폴더 내의 `extension/dist` 디렉토리를 선택하여 로드합니다.
    - **주의**: 빌드 전인 `extension` 폴더가 아니라, 빌드 결과물이 담긴 `extension/dist` 폴더를 선택해야 합니다.

### 2단계: 데이터 캡처 시작

1.  분석하고자 하는 React 기반 웹사이트를 브라우저에서 엽니다.
2.  브라우저 우측 상단의 확장 프로그램 아이콘 리스트에서 **'ReactPerf Devtool'**을 확인합니다.
3.  웹사이트를 이용하면 자동으로 컴포넌트 렌더링 횟수, 렌더링 시간, API 응답 속도 등이 수집됩니다.

### 3단계: 대시보드 분석

1.  [http://localhost:3000/dashboard](http://localhost:3000/dashboard) (또는 로컬 서버 주소)로 접속합니다.
2.  수집된 데이터가 있는 경우 **'Performance Sessions'** 목록에 새로운 세션이 나타납니다.
3.  각 세션을 클릭하여 상세 지표(컴포넌트별 렌더링 통계 등)를 분석합니다.

## 3. 관리자 계정 생성 (필요 시)

대시보드 접근을 위해 로그인이 필요하지만 계정이 없는 경우, 다음 명령어를 통해 환경 변수(`.env`)에 정의된 관리자 계정을 생성할 수 있습니다.

```bash
# .env 파일에 ADMIN_EMAIL, ADMIN_PASSWORD가 설정되어 있어야 합니다.
npx prisma db seed
```

---

> [!TIP]
> 데이터가 실시간으로 반영되지 않는다면 브라우저를 새로고침하거나 확장 프로그램의 연결 상태를 확인해 주세요.
